# Task 03 — ООП: интерфейсы и паттерн Strategy

## Задача
Реализовать фильтрацию задач через сменяемые стратегии.

## Что изучалось

- `interface` как контракт поведения
- `@FunctionalInterface` — интерфейс с одним методом
- Паттерн **Strategy** — поведение передаётся снаружи через интерфейс
- `final` поля — неизменяемость после создания объекта
- Предвычисление в конструкторе вместо повторных вычислений в методе

## Ключевой вывод
`TaskSearcher` не знает КАК фильтровать — только ЧТО делать с результатом.
Новый фильтр = новый класс, без изменения существующего кода (OCP из SOLID).
